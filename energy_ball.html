從《七龍珠》的能量球，到真實宇宙中的恆星

本文件整理了一連串關於動畫特效、CSS 視覺效果、科普影像， 以及「真實恆星長什麼樣」的討論。 核心問題是：

為什麼七龍珠裡的能量球，和 YouTube 科普頻道裡的恆星看起來幾乎一模一樣？

一、七龍珠的能量球是用 CSS 做的嗎？

不是。

  • 《七龍珠》是動畫作品，製作流程包含手繪原畫與數位後期合成
  • 沒有 HTML、沒有瀏覽器、也沒有 CSS
  • 使用的是動畫與影像軟體（手繪、上色、發光、疊加）
七龍珠能量球 和 CSS 在工具層完全無關。

二、那為什麼看起來「很像 CSS 漸層」？

因為 CSS 漸層並不是憑空發明的，它只是把早就存在於 繪畫、動畫、設計中的視覺原理「數學化」。

  • 中心亮、外圍暗 → 放射狀漸層
  • 模糊後變亮 → 發光（Glow）
  • 邊緣不規則 → 雜訊 / 手繪抖動

動畫師、設計師、前端工程師， 只是用不同工具表現同一套視覺邏輯。

三、YouTube 科普頻道裡的「恆星」是真的樣子嗎？

也不是。

多數科普影片中的恆星畫面，屬於：

  • 資料視覺化（visualization）
  • 偽彩色（false color）
  • 對比與亮度被大幅拉伸
  • 或乾脆是 CG / 動畫示意
那些畫面是「為了讓人看懂」，不是「你真的會看到的樣子」。

四、現實中的恆星實際上長什麼樣？

以太陽為例（唯一能近距離觀測的恆星）：

  • 看起來是一個非常穩定、邊緣清楚的光圓盤
  • 不會像火焰一樣翻騰
  • 亮到肉眼幾乎看不到細節
  • 夜空中「閃爍」其實是地球大氣造成的

五、為什麼動畫、科普、特效會「撞臉」？

這是一種視覺收斂現象。

只要你要表現：
  • 高能量
  • 高溫
  • 球狀天體
  • 不可直視的亮度

人類幾乎一定會畫成： 「中心亮、外圍模糊、有翻湧感的發光球體」

所以你看到的不是「同一個軟體的痕跡」， 而是同一套人類共用的視覺語言。

六、總結

  • 七龍珠能量球不是 CSS 做的
  • 科普影片中的恆星也不是恆星的真實外觀
  • 它們相似，是因為都在使用「高能天體的視覺隱喻」
  • CSS、Photoshop、動畫只是不同實作方式
動畫不是物理，視覺化不是照片，但它們幫助我們理解世界。
